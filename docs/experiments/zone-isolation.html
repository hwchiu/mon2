Experiment 01

Zone1 / Zone2 Host Isolation

這一輪不是在玩 cluster mesh，也不是 workload identity。它只做一件事： 驗證同一套 Calico control plane，能不能用 HostEndpoint labels 同時管理 standalone VM 和 cluster-2 nodes as external hosts，並做出清楚的 zone-based host firewall。

實驗定義很直接：legacy-01 與 cluster-2 三個節點都是 zone1， legacy-02 是 zone2。目標是 zone1 -> zone1 允許，而 zone1 <-> zone2 全部阻擋。

Policy 設計

Allow Rule
zone1 -> zone1

針對 selector zone == "zone1" 的 HostEndpoints，允許同 zone 來源存取 TCP 18080。

Deny Rule
zone2 -> zone1

對所有 zone1 HostEndpoints，顯式拒絕來自 zone2 的 TCP 18080。

Deny Rule
zone1 -> zone2

對 zone2 HostEndpoints，顯式拒絕來自 zone1 的 TCP 18080。

觀察與限制

這輪已經證明的事

Calico 不必接管 cluster-2 的 CNI，也能把 cluster-2 nodes 與 standalone VMs 一起納入同一套 host-level zoning。

這輪刻意沒做的事

沒有直接下全埠 zone firewall，因為那會同時碰到 SSH、k3s control-plane 與 Cilium 自身流量。

下一步

把目前只針對 TCP 18080 的 zoning，擴成多 port、多 profile，或開始試更接近 identity-based firewall 的規則模型。

這一頁現在是「結果頁」而不是「原始 dump」。目的不是把所有 YAML 直接貼上來，而是讓你先一眼看懂： 誰被分到哪個 zone、policy 怎麼下、最後測試是否符合預期。
