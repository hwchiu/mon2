Calico 軌道 · 實驗 01

Zone1 / Zone2 主機隔離

這是 Calico 軌道目前最完整的一頁結果。它先把拓樸講清楚，再把矩陣與結論講清楚， 讓讀者快速看見為什麼最後得到 zone1 → zone1 允許，以及 TCP 18080 上的跨區拒絕。

結果
7 / 7
連線探測與 policy 預期一致
允許面 — 所有 zone1 內部的 18080 探測都成功。
拒絕面 — zone1 ↔ zone2 兩個方向都被完整阻擋。
安全性 — 只測 host service，不碰 SSH、控制平面與 Cilium datapath。

拓樸

只保留三層：控制平面、policy engine、受管理 estate。刻意省略 jumpbox、placeholder nodes 與原始 YAML， 讓讀者直接看懂 policy 如何從 cluster-1 流向 zone1 與 zone2。

Policy 設計

允許規則
zone1 → zone1

對所有 zone == "zone1" 的 HostEndpoint，允許同區來源在 TCP 18080 存取。

拒絕規則
zone2 → zone1

明確拒絕所有 zone2 來源在 TCP 18080 打到 zone1 HostEndpoint。

拒絕規則
zone1 → zone2

明確拒絕所有 zone1 來源在 TCP 18080 打到 zone2 HostEndpoint。

結論

這輪證明了什麼

Calico 能在不接管 cluster-2 CNI 的前提下，把 host-level zoning 一致下發到 cluster-2 nodes 與 standalone VMs。

刻意收斂的範圍

只測 TCP 18080，不做整區封鎖，避免不小心打壞 SSH、k3s control plane 或 Cilium 本身。
