Experiment 01

Zone1 / Zone2 Host Isolation

這個實驗驗證單一 Calico control plane 是否能用 HostEndpoint labels，在兩種不同 estate 上 同步下發 zone-based host policy： 一邊是 standalone VM，另一邊是「第二個 k3s cluster 的節點，但全部當 external hosts」。

實驗目標

Scope

受管對象

legacy-01、legacy-02、以及 cluster-2 三個 nodes。

Intent

Zone 規則

zone1 -> zone1 允許；zone1 <-> zone2 禁止。

Safety

驗證範圍

先限定在 host test service 的 TCP 18080，不碰 k3s/Cilium 管理流量。

Zone 劃分

Zone 成員 位址 備註
zone1 legacy-01 10.70.20.20 standalone VM，Felix-only external host。
zone1 hep-0605-0534-k3s2-server-0 10.70.30.10 cluster-2 control-plane node，但在 Calico 眼中是 external host。
zone1 hep-0605-0534-k3s2-agent-01 / agent-02 10.70.30.11-12 cluster-2 worker nodes，同樣走 HostEndpoint 模式。
zone2 legacy-02 10.70.20.21 standalone VM，保留 restricted profile 並疊加 zone label。

實驗架構

套用設定

Calico manifests

  • manifests/calico/zone-experiment/00-zone-hostendpoints.yaml
  • manifests/calico/zone-experiment/10-allow-zone1-to-zone1-http.yaml
  • manifests/calico/zone-experiment/20-deny-zone1-to-zone2-http.yaml

實驗腳本

  • scripts/run-zone-isolation-experiment.sh
  • 在五台 external hosts 上建立 lab-zone-http.service
  • HTTP 測試埠：18080
./scripts/run-zone-isolation-experiment.sh \
  --cluster1-kubeconfig /home/ubuntu/.kube/hep-0605-0534-k3s.yaml \
  --jumpbox azureuser@20.110.199.106

核心 policy:
- selector: experiment == "zone-isolation" && zone == "zone1"
- selector: experiment == "zone-isolation" && zone == "zone2"
- test port: TCP 18080

觀察

這一輪實驗已經證明：在目前的 lab 拓樸下，Calico 不需要接管第二個叢集的 CNI， 只要把 cluster-2 nodes 視為 external hosts，就能和 standalone VMs 一起被單一 control plane 用 label selector 做同一套 host-level zoning。

同時，這仍然只是第一輪驗證。它目前只覆蓋專用測試埠 TCP 18080，還沒有擴大到多埠、 多 profile、或真正的 identity-based firewall 規則。
