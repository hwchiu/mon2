Experiment 01

Zone1 / Zone2 Host Isolation

這頁現在只做兩件事：先把拓樸畫清楚，再把結果畫清楚。 目標很單純，讓你一眼看出誰在 zone1、誰在 zone2， 以及為什麼最後會得到 zone1 -> zone1 allow、zone1 <-> zone2 deny。

Outcome
7 / 7
connectivity probes matched the policy intent
Allow side

zone1 內部流量在測試埠 18080 全數成功。

Deny side

zone1 -> zone2 與 zone2 -> zone1 全數被阻擋。

Safety scope

只驗證 host service，不去碰 jumpbox SSH、k3s control-plane 或 Cilium datapath。

實驗拓樸圖

圖只畫三個層次：control plane、policy engine、managed estates。沒有把 jumpbox、placeholder nodes、或原始 YAML 細節塞進圖裡，所以閱讀重點只剩 policy 如何從 cluster-1 下發到 zone1 與 zone2。

Policy 設計

Allow Rule
zone1 -> zone1

對 zone == "zone1" 的 HostEndpoints，允許同 zone 來源存取 TCP 18080。

Deny Rule
zone2 -> zone1

對所有 zone1 HostEndpoints，顯式拒絕來自 zone2 的 TCP 18080。

Deny Rule
zone1 -> zone2

對 zone2 HostEndpoints，顯式拒絕來自 zone1 的 TCP 18080。

結論

這輪已經證明的事

Calico 不用接管 cluster-2 的 CNI，也能把 cluster-2 nodes 和 standalone VMs 一起納入同一套 host-level zoning。

這輪刻意限制的範圍

只做 TCP 18080，不直接下全埠 zone firewall，避免誤傷 SSH、k3s control-plane 與 Cilium 自身流量。
