關鍵判讀

Cluster node 路徑

host firewall 最終有被啟用，`policy-cidr-match-mode=nodes` 也有補上，但在 pinned 1.17.6 的這個 host-policy 形狀下，cross-zone deny 仍沒有成為可觀察到的有效封鎖。

Linux deprecated 路徑

external workload registration 存在，cluster identity 也存在，但 Linux VM 本地 `policy get []`、`Host firewall: Disabled`，所以它不能算通過。

Windows 分支

Windows VM 與 IIS 都正常，這不是 bootstrap failure。真正缺的是一條可信、官方支援的 standalone Cilium-managed Windows path，因此這一列應歸類為產品限制，而不是 lab error。

主要 live findings

Cluster 端

  • 必須額外啟用 hostFirewall.enabled=true 與 devices=eth0。
  • 必須補上 policy-cidr-match-mode=nodes 才能讓 peer node IP 出現 cidr:* identity。
  • 即使如此，host endpoint 的 BPF policy map 仍未呈現預期 deny entries。

Standalone 端

  • Linux deprecated path 需要先修正 cluster.id = 1 並重建 clustermesh。
  • Windows bootstrap template 只安裝 OpenSSH 與 IIS，並沒有安裝 standalone Cilium agent。
  • Windows row 的正確分類是 No official standalone Windows path found，不是 infrastructure bring-up failure。
