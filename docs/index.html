Host Firewall Validation Lab

這個 repo 不是單一產品展示頁，而是一個持續演進的驗證入口網站。它用同一套 Azure lab 比較不同控制平面下的 host / standalone firewall story，並把結果分成清楚的 verdict， 讓讀者能快速區分成功、限制與失敗。

這個入口網站在做什麼

驗證對象

聚焦在 Kubernetes nodes、Linux standalone VM、Windows standalone VM 三類 target。

驗證方式

先維持範圍可控，再用單一 port、單一服務、明確 matrix 去驗證 allow / deny 行為。

判讀原則

產品限制、deprecated path、lab/config error 分開記錄，不混成一個模糊結論。

如何判讀結果

Supported pass

代表主要驗證路徑在宣告的 matrix 下運作正常，且沒有把環境打壞。

產品限制

代表環境本身可達、bootstrap 沒壞，但沒有找到官方支援或可成立的產品路徑。

驗證失敗

代表路徑有被正確嘗試，但結果不符合預期，或關鍵 policy 沒有真正生效。
