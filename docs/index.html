第二階段目標架構

下一階段是第二個 Kubernetes 叢集，也就是你提到的 TKS / cluster-2 目標。 文件會先把它畫成 target architecture，但目前還不把它當成已完成狀態。

已定義

Target shape

  • 新增第二個 Kubernetes 叢集。
  • 第二個叢集仍採用 Cilium 當 CNI。
  • Calico 只保留 policy/identity 相關元件。
  • 目標是把多叢集 workload 與 VM 放進同一個 selector 模型。

  • 因此首頁先明確區分 current state 與 target state。
